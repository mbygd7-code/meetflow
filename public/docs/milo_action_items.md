# 📋 Milo Action Items — 회의록 & 액션아이템 품질 규칙

> 회의 종료 시 생성하는 4섹션 회의록(decisions/discussions/deferred/action_items)과
> 액션아이템 추출의 **품질 기준 & 판단 사례집**.

---

## A. 4섹션 분류 기준

### A-1. `decisions` (결정사항)
**조건 (모두 충족)**:
- 명시적 합의 신호 존재 ("결정합시다", "그걸로 갑시다", "동의합니다")
- 반대 의견이 철회되거나 부재
- 누가/무엇을/어떻게 중 최소 2개가 명확

**포함 예시**:
- ✅ "B안으로 확정, 지우 님이 담당"
- ✅ "다음 스프린트에 포함시키기로 결정"

**제외 예시**:
- ❌ "B안이 좋아 보이네요" (개인 의견, 합의 아님)
- ❌ "나중에 결정하죠" → `deferred`로

### A-2. `discussions` (논의사항)
**조건**:
- 의견이 교환되었으나 결론 없음
- 복수 관점이 공존
- 결정도 연기도 명시되지 않음

**포함 예시**:
- ✅ "타겟 고객 세그먼트에 대해 원장 vs 교사 관점 대비"
- ✅ "기술 부채 해결 우선순위에 대한 의견 교환"

### A-3. `deferred` (연기된 항목)
**조건**:
- 명시적 연기 표현 ("다음에", "보류", "나중에", "이번엔 패스")
- 재논의 시점이 특정되거나 암시됨

**포함 예시**:
- ✅ "가격 정책은 다음 분기 회의에서 재논의"
- ✅ "v2 기능 스펙은 MVP 출시 후 논의 보류"

**필수 필드**: `title` + `reason` (왜 연기되었는지)

### A-4. `action_items` (실행 과제)
**조건 (모두 충족)**:
- 실행 가능한 동작 (누가 하면 결과가 나오는 것)
- 어젠다 논의에서 도출된 후속 작업
- 본 회의에서 완결되지 않음

**제외 예시**:
- ❌ "생각해보겠습니다" (행위 불명확)
- ❌ "좋은 의견이네요" (결정·실행 아님)

## B. 액션아이템 구성 5요소

```json
{
  "title": "A안 기획서 초안을 작성한다",
  "assignee_hint": "박지우",
  "priority": "high",
  "due_hint": "다음 주 금요일",
  "source_turn": "메시지 #27 — 지우: '기획서는 제가 맡을게요'"
}
```

### B-1. `title` — 동사로 시작
- ✅ "경쟁사 조사 리포트를 작성한다"
- ❌ "경쟁사 조사" (명사구, 동작 불명확)
- **원칙**: 읽는 사람이 "무엇을 하면 되는지" 즉시 이해

### B-2. `assignee_hint` — 담당자 추정 우선순위
1. **본인 선언** (최우선): "제가 할게요", "저한테 넘겨주세요"
2. **지명**: "지우 님이 해주세요" → 지우 님이 암묵적 수락
3. **역할 추정**: "기획 파트에서 정리" → 해당 역할자 힌트
4. **불명확**: `"미정"` (절대 임의 지정 금지)

### B-3. `priority` 결정 트리
```
[이 태스크가 지연되면 무엇이 막히는가?]
    │
    ├─ 다른 작업이 시작조차 불가 (블로커) ───→ urgent
    ├─ 법적 리스크 / 보안 이슈 ────────────── urgent
    ├─ 다음 마일스톤 전 필수 (1주 이내) ─────→ high
    ├─ 다음 스프린트(1~2주)에 필요 ──────── medium
    ├─ 백로그, 시점 여유 있음 ──────────────→ low
    └─ 불명확 → medium (기본값)
```

### B-4. `due_hint` — 자연어 허용
- ✅ "다음 주 금요일", "이번 달 말", "다음 스프린트"
- ✅ "2026-04-30" (명시적 날짜)
- ⚠️ "최대한 빨리" — 허용하되 `priority: high` 이상 필요
- ❌ 임의로 날짜 지어내기 금지

### B-5. `source_turn` — 근거 턴
- 회의 내 어느 발언에서 도출되었는지 명시
- 추후 검증·이의제기 시 추적 가능
- 형식: 메시지 번호 또는 발화자 + 핵심 문구

## C. 액션아이템 vs 논의사항 구분 사례

| 발언 | 분류 | 이유 |
|------|------|------|
| "철수 님이 A안 기획서 금요일까지 작성" | ✅ action_item | 실행자+행위+기한 명확 |
| "A안이 좋아 보이네요" | ❌ → discussion | 의견, 실행 아님 |
| "B안으로 가기로 합시다" | ✅ decision | 결정, but 후속 task 별도 |
| "나중에 검토해봅시다" | ❌ → deferred | 연기 |
| "지표 뽑아서 공유드릴게요" | ✅ action_item | 본인 선언 |
| "성과가 아쉽네요" | ❌ → discussion | 감정 표현 |
| "다음 미팅까지 경쟁사 3곳 벤치마크" | ✅ action_item | 범위+기한 명확 |
| "이거 누가 할까요?" | ❌ → discussion | 담당자 미정, deferred도 아님 |

## D. 애매한 경우 처리 규칙

### D-1. 담당자 불명확
- 회의 참여자 중 추정 어려움 → `assignee_hint: "미정"`
- **절대 임의 지정 금지**: 참여자가 아닌 사람을 지정하면 혼란 유발
- 회의록 생성 후 사용자가 수동 배정하도록 UI 유도

### D-2. 기한 불명확
- 구체 시점 미언급 → `due_hint: "미정"`
- `priority`만으로도 긴급도 전달 가능 → 임의 날짜 삽입 금지

### D-3. 동일 액션 중복
- 같은 태스크가 여러 번 언급 → 1건으로 통합, 가장 구체적인 선언 채택

### D-4. 조건부 액션
- "만약 A라면 B를 한다" → 조건이 충족될 때만 action_item
- 조건 미충족 상태면 `deferred`로 분류

### D-5. 암묵적 액션
- "당연히 리뷰 돌려야죠" 같은 암묵 언급 → 명시적이지 않으면 제외
- 사용자가 수동 추가할 수 있음

## E. `milo_insights` 작성 지침

회의록의 마지막 필드. 2~3문장의 **메타 관찰**.

### E-1. 포함할 내용
- 회의 전반의 흐름 (집중/분산/교착 여부)
- 의사결정 속도 (빠름/지연/회피)
- 추후 주의할 패턴

### E-2. 좋은 예시
- ✅ "기획 방향 자체에는 합의가 있었으나 세부 일정에서 의견이 갈렸습니다. 다음 회의 전 스펙 문서 초안이 있으면 속도가 붙을 것으로 보입니다."
- ✅ "의사결정이 3건 있었으며 모두 담당자가 지정되었습니다. 팔로업 주기만 합의되면 진행에 무리 없습니다."

### E-3. 금지
- ❌ 개인 평가: "지우 님이 의견이 없으셨습니다"
- ❌ 단순 요약 반복: "A안, B안을 논의했습니다" (이미 discussions에 있음)
- ❌ 감정 표현: "좋은 회의였습니다"

## F. 품질 체크리스트

회의록 생성 직전 자체 점검:

### 분류 정확도
- [ ] decisions에 합의되지 않은 의견이 섞이지 않았는가?
- [ ] action_items에 "생각해본다"류 비실행 항목이 없는가?
- [ ] deferred에 reason 필드가 있는가?

### 액션아이템 품질
- [ ] 모든 title이 동사로 시작하는가?
- [ ] assignee_hint가 임의 지정되지 않았는가?
- [ ] priority 결정이 일관된 기준(B-3)을 따랐는가?
- [ ] source_turn이 검증 가능한가?

### 종합
- [ ] milo_insights가 사실 기반이고 감정 평가가 없는가?
- [ ] 전체 분량이 원 회의 대비 적절한가? (과다 요약도 과소 요약도 ❌)

---

*회의록은 미래의 자신과 팀에게 남기는 메모다. 정확한 분류가 정확한 실행을 만든다.*
